Your payroll connection receives the final time-segment output generated by each sync run.

Current payroll provider:

- `ADP`

## Steps

1. Open **Settings**.
2. In the **Payroll** section, choose your payroll provider.
3. Enter the required connection fields.
4. Save the connection.
5. Run the connection test.

For ADP, the current connection form requires:

- client ID
- client secret
- client certificate PEM
- private key PEM

![Payroll section with provider selection and connection status](./images/payroll-section-provider-select.png)

![Payroll connection form with required fields ready to save](./images/payroll-connection-modal.png)

## Best practices

- use an organization-owned integration credential set
- validate the provider in a dry run before switching to live sync
- keep certificate and key rotation ownership clear inside your team

## When a payroll connection is required

The dashboard, Jobs page, and manual driver sync flow all expect an active payroll connection. If the payroll connection is missing, HOSSync will stop the work from being queued.
